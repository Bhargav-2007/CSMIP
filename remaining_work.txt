SECTION 5 — KNOWN BUGS (Not yet fixed)
═══════════════════════════════════════════════════════════

  BUG-1: Dashboard axios calls have no .catch() fallbacks
    → If any one of the 4 API calls fails, the component silently breaks
    → Fix: Add .catch(() => []) to each axios call in Dashboard.jsx

  BUG-2: Schemes API endpoint is /api/services/schemes (note: under /services/)
    → This conflicts with the /:slug dynamic route in services.js
    → /services/schemes will try to find a service with slug="schemes" first
    → Fix: Move schemes to its own route file or rename path to /api/schemes

  BUG-3: Assistant file upload calls /api/api/documents/upload (double /api/)
    → Line 77 in Assistant.jsx: `${API_URL}/api/documents/upload`
    → API_URL already includes /api — remove the extra /api from this path

  BUG-4: Complaint and RTI load() called with empty token on mount
    → useEffect runs before auth token is restored from localStorage
    → Add token as dependency: useEffect(() => { if(token) load(); }, [token])
